nacos v2

进入管理界面的地址是http://localhost:8848/nacos,默认管理员用户名和密码都是nacos，v2要求DataId严格带后缀名，比如okrm-gateway.yml，nacos-config-exampe.properties的后缀是严格要求和文件格式匹配，否则连接不上。

![image-20260314110205844](/Users/pe7er/Library/Application Support/typora-user-images/image-20260314110205844.png)

```
# nacos-config-example.properties
spring.cloud.nacos.config.serveraddr=127.0.0.1:8848
spring.cloud.nacos.config.prefix=PREFIX
spring.cloud.nacos.config.group=GROUP
spring.cloud.nacos.config.namespace=NAMESPACE
```

```yaml
# okrm-gateway.yml
spring:
  cloud:
    nacos:
      config:
        server-addr: http://[::1]:8848
        prefix: PREFIX
        group: GROUP
        namespace: NAMESPACE
    gateway:
      routes:
        - id: user-service
          uri: http://[::1]:9000
          predicates:
            - Path=/user-service/**


logging:
  level:
    org.springframework.cloud.gateway: DEBUG
    com.alibaba.nacos: DEBUG
    com.uhomeup.gateway: DEBUG

```

nacos v3

进入管理界面的地址是http://localhost:8849/,默认管理员用户名和密码都是nacos，v3 不要求 DataId 带后缀名了



```yaml
# okrm-gateway
server:
  port: 8000
spring:
  application:
    name: gateway-service
  cloud:
    nacos:
      config:
        server-addr: localhost:8848
        group: DEFAULT_GROUP
        file-extension: yaml
        username: nacos
        password: nacos
      discovery:
        server-addr: localhost:8848
        username: nacos
        password: nacos
    gateway:
      server:
        webflux:
          discovery:
            locator:
              enabled: true
          routes:
            - id: user-service
              uri: lb://user-service
              # uri: http://localhost:9000
              predicates:
                - Path=/user-service/**
  level:
    org.springframework.cloud.gateway: DEBUG
    com.alibaba.nacos: DEBUG
    reactor.netty: DEBUG

```
